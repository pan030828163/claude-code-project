# 工具链从零搭建

## 今日掌握

（无前日复盘，从今日起始状态开始）

今天是第一次系统性地把 Claude Code 的开发工具链跑通。此前对 AI 编程助手的认知停留在"对话式问答"，今天实际动手后发现它更像一个可扩展的代理操作系统——Skills 是功能模块，MCP 是硬件驱动，Git 是版本记忆。

上午从 Excel 技能入手，搜索对比了两类方案：`anthropics/skills@xlsx`（90K 安装量，文件级读写）和 `sbroenne/mcp-server-excel@excel-mcp`（1.1K 安装量，COM 接口实时操控）。这个差异触发了一个认知：同样是"操作 Excel"，底层实现路径完全不同——前者依赖 openpyxl 等 Python 库直接解析 xlsx 结构，后者通过 Windows COM 进程间通信控制 Excel 应用程序实例。两者适用场景互补：批量处理走文件级，需要触发展示或公式计算的走 MCP。

安装过程中暴露了网络拓扑问题：`npx skills` 能成功克隆 GitHub 仓库，但 `git push` 直连 443 端口超时。排查发现 npm 配置了 npmmirror 国内镜像，而 git 没走代理。解决思路从 HTTPS 切换到 SSH（端口 22），配合刚生成的 ED25519 密钥对完成了认证。这个环节踩了三个坑——host key 验证需手动加入 known_hosts、公钥粘贴时换行符导致 GitHub 校验失败、ssh-agent 未启动导致密钥无法加载。排查过程对 SSH 握手协议有了更具体的感觉。

最终完成了 git init → .gitignore → commit → remote add → push 全流程，311 个文件成功推送到 GitHub。回头想，今天不是"学了一个工具"，而是把 AI Agent、技能包管理、版本控制这三个原本独立的系统串成了可复用的工作流。

## 相关知识库

### Skills 生态系统

Claude Code 的 Skills 是可插拔的能力扩展模块。每个 Skill 包含一个 `SKILL.md` 描述文件和可选的脚本、参考文档、模板资源。安装来源可以是 GitHub 仓库的特定子目录（`@skill-name` 语法指定分支或标签），安装级别分项目级（`--project`）和全局级（`-g`）。全局安装的实际路径在 `~/.agents/skills/`，项目级在 `.agents/skills/`。Skills CLI（`npx skills`）提供 find、add、check、update 四个核心命令，底层通过 npm registry 分发（国内用户建议配置 npmmirror 镜像）。

- GitHub: https://github.com/anthropics/skills
- 技能市场: https://skills.sh/

### SSH 密钥认证

SSH 密钥对（这里用 ED25519 算法，比 RSA 更短更安全）用于替代密码进行远程认证。私钥（`id_ed25519`）留在本地，公钥（`id_ed25519.pub`）上传到 GitHub 的 Settings → SSH and GPG keys。连接时 GitHub 用公钥加密一段挑战，只有持有对应私钥的客户端能解密应答，完成身份验证。ssh-agent 是密钥缓存进程，避免每次连接都输入私钥密码。known_hosts 文件存储已信任主机的公钥指纹，防止中间人攻击。

- GitHub SSH 文档: https://docs.github.com/en/authentication/connecting-to-github-with-ssh
- ED25519 规范: https://ed25519.cr.yp.to/

### Git 版本控制基础

`git init` 在当前目录创建 `.git` 隐藏目录，开始版本追踪。`git add` 将修改加入暂存区，`git commit` 将暂存区内容固化为一个快照对象（commit object）。`git remote add origin <url>` 关联远程仓库，`git push -u origin master` 将本地 master 分支推送到远程并建立追踪关系。`.gitignore` 文件用 glob 模式排除不需要追踪的文件（构建产物、缓存、密钥等）。

- Git 官方文档: https://git-scm.com/doc
- GitHub Git 指南: https://github.com/git-guides

### MCP（Model Context Protocol）

MCP 是 Anthropic 提出的开放协议，定义了大模型与外部工具/数据源之间的标准通信接口。MCP 服务器是独立的进程，通过 JSON-RPC over stdio 或 HTTP 暴露工具能力。Claude Code 作为 MCP 客户端，在需要时调用服务器提供的具体工具。excel-mcp 就是基于此协议，通过 Windows COM 接口桥接到 Excel 应用程序，实现单元格读写、格式设置、图表操作等能力。

- MCP 规范: https://github.com/modelcontextprotocol/specification
- excel-mcp 仓库: https://github.com/sbroenne/mcp-server-excel

### npm 镜像机制

npm 默认从 `registry.npmjs.org` 拉取包，国内网络环境下延迟高且可能丢包。npmmirror.com（原淘宝镜像）是国内维护的完整 npm 镜像，每 10 分钟同步一次。配置方式：`npm config set registry https://registry.npmmirror.com`。npx 执行远程包时也会走这个 registry，所以 `npx skills` 能正常工作而 `git push` 直连失败——两条链路走了不同的网络出口。

- npmmirror: https://npmmirror.com/
- npm 官方文档: https://docs.npmjs.com/
