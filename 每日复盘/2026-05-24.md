# 初识 AI 代理

## 今日掌握

无前日复盘，这是 Claude Code 安装首日，一切从零起步。

最开始的问题很朴素：指挥这个工具写代码需要提供什么。得到的答案也很简单——自然语言描述想做什么加文件路径，不必拘泥特殊格式。这个低门槛的交互模型打破了此前对命令行工具必须有严格语法的认知，但也让我意识到另外一件事：工具本身不难用，难的是知道它能力的边界在哪里。

下午围绕两个方向做了实际探索。第一个是 MCP 协议——花了不少时间整理了一篇完整的知识文档，从协议架构（客户端-服务器模型、JSON-RPC 通信、stdio 与 SSE 双传输模式）到实际配置案例都梳理了一遍。把 MCP 理解为 AI 领域的 "USB 接口" 这个类比很有效，它解决的正是大模型连接外部工具时"每个数据源都要单独写对接代码"的碎片化问题。Claude Code 内置的 Playwright MCP 就是这套协议的直接体现——浏览器自动化能力通过 MCP Server 暴露为可调用的工具。

第二个方向是 Hyperframes 视频框架。用它搭了一个 Swiss Grid 风格的动画合成项目，包含了 multiple compositions、GSAP 时间线动画、场景过渡和字幕系统。这个过程中接触到一些此前没深入想过的概念：动画的时序可控性（暂停/跳转/渲染）、声明式属性约定（data-start / data-duration / data-track-index）如何替代脚本硬编码、以及 `window.__timelines` 注册机制怎样让框架层统一控制所有动画实例。

另外一口气装了 15 个项目级 Skills——animejs、gsap、three.js、lottie、waapi、tailwind 等前端动画技术栈。这些 Skills 的作用不是教我基础语法，而是提供 Hyperframes 框架内的适配模式，确保动画代码在渲染管线里行为正确。

效果上，从"不知道能做什么"变成了"有一个可运行的多场景视频合成项目"，工具链初步跑通。

## 明日拓展

MCP 文档是整理完了，但实际配通过几个 MCP Server？目前只用了内置的 Playwright，外部社区 Server 还没碰过。比如文件系统 Server 的路径隔离是怎么做的，会不会有越权读写的风险？另外 Hyperframes 项目的 `<audio>` 同步机制在 seek 跳转时精度如何，会不会出现音画不同步？Skills 安装走了 `npx skills add`，但它是从 npm registry 下载还是直连 GitHub——如果是后者，国内网络能稳定访问吗？

## 相关知识库

### MCP（Model Context Protocol）

MCP 是 Anthropic 于 2024 年 11 月发布的开放协议，定位于大模型与外部工具之间的标准化通信层。它采用客户端-服务器架构：MCP Host（如 Claude Desktop）内嵌 MCP Client，与一个或多个 MCP Server 建立一对一长连接。Server 可提供三种能力——Resources（只读数据，如文件内容）、Tools（可执行函数，需用户批准）和 Prompts（预置模板）。通信格式统一为 JSON-RPC 2.0，本地走 stdio，远程走 SSE + HTTP。核心价值是用一套协议取代"每个工具单独对接"的集成碎片。

- MCP 规范仓库: https://github.com/modelcontextprotocol/specification
- 官方 Server 列表: https://github.com/modelcontextprotocol/servers
- Anthropic 发布博客: https://www.anthropic.com/news/model-context-protocol

### Hyperframes

Hyperframes 是基于 HTML/CSS/JS 的视频合成框架，核心理念是将浏览器渲染能力用于视频制作。项目结构以 `index.html` 为根组合（root timeline），`compositions/` 目录存放子场景，通过 `data-composition-src` 属性动态引入。所有时间元素需声明 `data-start`、`data-duration` 和 `data-track-index` 三个属性，可见元素附加 `class="clip"` 用于框架级可见性控制。动画引擎通过 `window.__timelines` 全局注册 GSAP 时间线，框架借此统一控制暂停、跳转和逐帧渲染。与视频编辑软件的本质差异在于：它是声明式代码驱动，而不是时间轴拖拽。

- Hyperframes 文档: https://hyperframes.heygen.com/introduction
- 机器可读索引: https://hyperframes.heygen.com/llms.txt

### GSAP（GreenSock Animation Platform）

GSAP 是 JavaScript 高性能动画库，核心抽象是 Tween（单元素属性动画）和 Timeline（多 Tween 的时间编排）。gsap.to/from/fromTo 控制起止状态，Timeline 支持位置参数（绝对/相对/标签）实现复杂序列编排。在 Hyperframes 中所有 Timeline 必须以 `paused: true` 创建并注册到 `window.__timelines`，框架通过 `hf-seek` 事件驱动时间轴跳转来保证动画帧精确可控。

- GSAP 官方仓库: https://github.com/greensock/GSAP
- GSAP 文档: https://gsap.com/docs/

### Skills 生态系统

Claude Code 的 Skills 是模块化的能力扩展包，本质是一组含有 SKILL.md 描述文件、脚本和参考文档的目录。安装来源为 GitHub 仓库（通过 `owner/repo@skill-name` 定位），分为项目级和全局级两个作用域。Skills CLI（`npx skills`）提供安装、查找、更新等命令。Skills 的价值不是替代官方文档，而是提供特定框架/工具的适配模式和最佳实践——比如 GSAP 的通用语法大家都知道，但在 Hyperframes 里如何注册、如何与渲染管线配合，这些就是 Skills 提供的上下文。

- Skills 市场: https://skills.sh/
- Anthropic Skills 仓库: https://github.com/anthropics/skills
